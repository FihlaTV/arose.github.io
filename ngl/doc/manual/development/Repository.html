Development is coordinated through github: http://github.com/arose/ngl. Please use the issue tracker there to report bugs or suggest improvements. Pull requests are also welcome.

Report issues

For your reports on the issue tracker, please provide a comprehensive description and include any information required to understand or reproduce the matter.

Development environment

To develop for the NGL Viewer you need a local copy of the source code, which you can obtain by forking the ngl repository on github. Read about how to fork a repository.

Pull request

Development of new features is done the *dev* branch, so unless you want to contribute a bug-fix for a release in the the *master* branch, your pull request must be against the *dev* branch.

First make sure you are in the *dev* branch.

git checkout dev

Create a new branch.

git checkout -b [name_of_your_new_branch]

Push your new branch to the origin of your local repository, probably on github.

git push origin [name_of_your_new_branch]

If in the meantime the main ngl *dev* branch has been updated you must update your branch accordingly. You can read more about how to sync a fork.

git fetch upstream

After fetching you need to merge the changes.

git merge upstream/dev

When you are satisfied with your changes, it is time to start a pull request via the github web interface as described here.

After a completed pull request you can delete the corresponding branch on your local filesystem.

git branch -d [name_of_your_new_branch]

And also delete the branch on github. Note the colon : in front of the branch name.

git push origin :[name_of_your_new_branch]
